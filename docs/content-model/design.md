# Content Model — Design

A well-formed content model node should:

- Define a single concept clearly in its description
- Link to related concepts within the content model
- Include concrete examples where applicable
- Be understandable without reading the reference implementation

Anti-patterns:
- Implementation-specific details (TypeScript types, function signatures)
- Duplicating content from the reference implementation
- Describing how the compiler works internally rather than what the content model requires
