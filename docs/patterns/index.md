# Patterns

Patterns are reusable approaches to common knowledge graph structuring challenges. They're not rules — they're conventions that work well in practice.

Each pattern describes when to use it, how it works, and what to watch out for. Current patterns include [collections](/patterns/collections), [linking](/patterns/linking), [placement](/patterns/placement), [progressive-disclosure](/patterns/progressive-disclosure), [authorship](/patterns/authorship), [frameworks](/patterns/frameworks), and [vibe-checking](/patterns/vibe-checking).
